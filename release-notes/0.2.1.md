- Changed the public release signing path so TermTile is now Developer ID signed instead of ad-hoc signed.
- This gives macOS Accessibility and Input Monitoring a stable Apple-issued code identity to grant, preventing the per-build permission break caused by ad-hoc cdhash signing.
- If you already installed an older ad-hoc build, you may need to remove TermTile from System Settings > Privacy & Security > Accessibility and Privacy & Security > Input Monitoring, then add it again once after installing this update. Future updates from this signing line should keep both grants.
- Notarization is still pending; this release is the immediate TCC-stability mitigation. Developer ID notarization is prepared, but Apple Notary processing did not return an accepted ticket during this cut.
