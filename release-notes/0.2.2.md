# TermTile 0.2.2

This is the notarized distribution fix.

- Public releases are now Developer ID signed, notarized by Apple, and stapled before the release zip
  is created.
- Release CI now blocks publishing unless Apple Notary returns `Accepted`, `stapler validate` passes,
  and Gatekeeper assessment succeeds.
- The app keeps the stable Developer ID identity introduced in 0.2.1, so Accessibility and
  Input Monitoring permissions remain tied to the same Apple code identity across updates.
- The v0.2.1 Notary queue delay has cleared; v0.2.2 is the first release intended to remove the
  Gatekeeper "Apple could not verify" warning for normal installs.
