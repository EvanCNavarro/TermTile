# TermTile 0.2.4

This is a release-hardening patch on top of the v0.2.3 stale-permission repair flow.

- Uninstall now uses the same scoped TCC repair primitive as **Repair Accessibility** and
  **Repair Input Monitoring**, clearing only TermTile's own Accessibility and Input Monitoring rows.
- The uninstall result now reports privacy-reset failures explicitly instead of claiming permission
  cleanup is always manual.
- Keeps the Developer ID signed, notarized, stapled, and Sparkle-signed release path.
