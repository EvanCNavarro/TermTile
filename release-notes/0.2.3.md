# TermTile 0.2.3

This is the stale permission repair release for people coming from older ad-hoc or development builds.

- Adds **Repair Accessibility** when macOS shows TermTile as enabled but the app still cannot arrange
  windows. The repair clears TermTile's stale TCC Accessibility row, opens the correct Settings pane,
  and lets the user approve the current signed app again.
- Adds **Repair Input Monitoring** for drag-reorder users with the same stale-grant problem. It resets
  only TermTile's ListenEvent TCC row, re-registers the current app, and opens Input Monitoring.
- Keeps the v0.2.2 Developer ID signed, notarized, and stapled release path. Normal updates from the
  stable Developer ID line should keep permissions; this repair is for older ad-hoc/dev history,
  duplicate copies, or moved apps that poisoned macOS's permission cache.
